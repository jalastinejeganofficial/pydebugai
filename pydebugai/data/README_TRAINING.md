## 🎯 What's Included

### Error Categories Covered

#### 1. **NameError** (123 samples)
- Undefined variables (`x`, `y`, `result`, etc.)
- Typos in built-in functions (`pritn`, `lenn`, `raange`)
- Missing imports (`os`, `sys`, `math`, etc.)
- Variable scope issues (free variables, globals)
- Common typos (`self`, `cls`, `kwargs`)

#### 2. **TypeError** (92 samples)
- Type mismatches in operations (`int` + `str`)
- Objects not callable (`'str' object is not callable`)
- Objects not iterable (`'int' object is not iterable`)
- Wrong argument counts (missing/extra args)
- String concatenation errors
- Indexing errors (wrong types)
- Unhashable types (`list`, `dict`)
- Attribute errors on wrong types

#### 3. **SyntaxError** (66 samples)
- Invalid syntax (missing colons, parentheses)
- Unexpected EOF
- String literal errors (EOL, quotes)
- Statement placement errors (`return`, `break`, `yield`)
- Starred expression errors
- Future import errors
- F-string errors
- Lambda and comprehension errors

#### 4. **AttributeError** (59 samples)
- NoneType attribute access (`.split()`, `.strip()`, etc.)
- Int attribute access (`.append()`, `.sort()`)
- Str attribute typos (`.lowr()`, `.uppr()`)
- Module attribute errors (`os.listdirr()`)
- Wrong method on type (`list.keys()`)

#### 5. **ValueError** (43 samples)
- Invalid literal conversions (`int('hello')`)
- Float conversion errors
- Unpacking errors (too many/few values)
- Math domain errors (`sqrt(-1)`)
- JSON decode errors
- Generator/coroutine reuse errors

#### 6. **ImportError** (37 samples)
- Missing modules (`numpy`, `pandas`, `requests`)
- Cannot import name errors
- Popular library imports (ML, web, data science)

#### 7. **IndexError** (31 samples)
- List/tuple/string index out of range
- Pop from empty list/stack/queue
- Empty container errors
- Assignment index errors

#### 8. **KeyError** (25 samples)
- Missing dictionary keys
- Common key names (`name`, `age`, `id`)
- Numeric key errors

#### 9. **RuntimeError** (26 samples)
- Collection modification during iteration
- Generator/coroutine errors
- Event loop errors
- Shallow/deep copy bugs

#### 10. **IndentationError** (15 samples)
- Unexpected indent/unindent
- Expected indented block errors
- Tab/space mixing errors
- Missing colon errors

#### 11-20. **Other Errors** (100+ samples)
- **ZeroDivisionError**: Division/modulo by zero
- **RecursionError**: Maximum recursion depth exceeded
- **OSError**: File not found, permission denied, network errors
- **UnicodeError**: Encode/decode errors
- **AssertionError**: Failed assertions
- **MemoryError**: Out of memory
- **OverflowError**: Math overflow
- **TimeoutError**: Operation timeouts
- **StopIteration**: Iterator exhausted
- **UnboundLocalError**: Local variable before assignment

### Special Categories

#### DSA Algorithm Errors (included in above counts)
- Sorting algorithm errors (quicksort, merge sort, heap sort)
- Binary search tree errors (traversal, insertion, deletion)
- Graph algorithm errors (DFS, BFS, Dijkstra)
- Dynamic programming errors (DP table, memoization)
- Linked list errors (traversal, node access)
- Stack/queue errors (underflow, overflow)
- Heap errors (index bounds, comparison)
- Hash table errors (collisions, load factor)
- Trie errors (children, end_of_word)
- Segment/Fenwick tree errors
- Greedy algorithm errors
- Backtracking errors

#### Common Beginner Mistakes (included in above counts)
- Using `=` instead of `==`
- Forgetting colons after statements
- Missing parentheses
- Quote mismatches in strings
- Boolean constant typos (`ture`, `flase`, `null`)
- Using keywords as variable names
- Mutable default argument bugs
- Variable scope issues
- Modifying while iterating
- Shallow vs deep copy confusion
- Integer division confusion
- Chained comparison misunderstandings
- List multiplication pitfalls

## 🔄 How to Regenerate Training Data

### Step 1: Generate New Examples

```bash
cd c:\new-model-ai\pydebugai\data
python generate_training_data.py
```

This will:
- Generate 600+ error examples
- Save to `error_patterns_comprehensive.json`
- Show category distribution

### Step 2: Merge with Existing Data

```bash
python merge_and_retrain.py
```

This will:
- Load original and comprehensive datasets
- Remove duplicates
- Validate minimum samples per category
- Save merged dataset to `error_patterns_merged.json`

### Step 3: Activate New Dataset

```bash
# Backup current dataset
Move-Item error_patterns.json error_patterns_backup_$(Get-Date -Format "yyyyMMdd").json

# Activate merged dataset
Copy-Item error_patterns_merged.json error_patterns.json
```

### Step 4: Retrain ML Classifier

```bash
# Go to project root
cd c:\new-model-ai

# Delete old model
Remove-Item pydebugai\models\classifier.pkl -Force
Remove-Item pydebugai\models\vectorizer.pkl -Force

# Run pydebugai to trigger retraining
pydebugai run tests/samples/demo.py
```

The classifier will automatically:
- Detect missing model files
- Train from `error_patterns.json`
- Save new model to `models/classifier.pkl`
- Report accuracy

## 🎓 Training Process Details

### ML Pipeline

1. **Text Vectorization**
   - TF-IDF with character n-grams (2-5)
   - Max 5000 features
   - Sublinear TF scaling
   - Character-level WB (word boundary) analyzer

2. **Classification**
   - Random Forest with 200 trees
   - Max depth: 12
   - Balanced class weights
   - Cross-validation (80/20 split)

3. **Evaluation**
   - Accuracy score reported
   - Stratified sampling ensures representation
   - Minimum 2 samples per category required

### Model Storage

- **Model**: `pydebugai/models/classifier.pkl`
- **Vectorizer**: `pydebugai/models/vectorizer.pkl`
- **Training Data**: `pydebugai/data/error_patterns.json`

## 🔧 Troubleshooting

### Issue: "Too few samples" warning

**Solution**: Run the generator to add more examples:

```bash
python generate_training_data.py
python merge_and_retrain.py
```

### Issue: Model accuracy is low

**Causes**:
- Too few training samples
- Imbalanced categories
- Overfitting on specific phrases

**Solutions**:
1. Add more diverse examples (aim for 50+ per category)
2. Balance category distributions
3. Use varied error message phrasings
4. Avoid duplicate patterns

### Issue: Training fails

**Check**:
1. scikit-learn installed: `pip install scikit-learn`
2. JSON format valid: `python -m json.tool error_patterns.json`
3. No empty categories
4. All category names match exactly

## 📚 Additional Resources

### Understanding TF-IDF

TF-IDF (Term Frequency-Inverse Document Frequency) converts text to numerical vectors:
- **TF**: How often a term appears in a document
- **IDF**: How rare the term is across all documents
- Character n-grams capture substrings like "not defined", "index out of"

### Why Random Forest?

- Handles high-dimensional sparse data well
- Robust to overfitting
- Provides confidence scores
- Fast inference
- Interpretable feature importance

### Alternative Approaches

Future improvements could include:
- **Deep Learning**: LSTM/GRU for sequence modeling
- **Transformers**: Fine-tune CodeBERT or similar
- **Hybrid**: Combine rules + ML (current approach)
- **Online Learning**: Continuously update from feedback
